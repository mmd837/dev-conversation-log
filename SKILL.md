# dev-conversation-log — 开发对话记录

按话题保存项目开发过程中用户和Claude的每一句话，原话不删减。

## 边界

**记录的内容：** 用户和Claude之间的对话文字（提问、回答、讨论、决定）
**不记录的内容：** 工具调用的过程（代码内容、文件读取内容、bash命令输出等）
**但必须记录：** 工具调用得出的结论（如"读完了代码，发现权重规则是..."）

## 触发方式

1. **主动模式**：用户说"记录这段对话"/"记录话题"/"保存对话"/"log this"
2. **建议模式**：我检测到一个话题讨论已结束（达成了结论或决定），主动问用户"要不要记录这段对话？"

## 执行步骤

### 第一步：确认项目路径

- 项目文件夹统一在 `C:\Users\liudi\Desktop\LDAI\claude code\<项目名>\`
- 如果用户指定了项目名，直接用
- 如果没有指定，根据当前对话上下文判断项目名，不确定则询问用户
- 检查是否存在 `开发对话记录\` 子文件夹，不存在则创建
- 读取已有文件确定下一个序号（已有 01、02 则下一个是 03）

### 第二步：确定话题范围

- 从当前对话中回溯，识别本次要记录的话题边界
- 如果一次对话中有多个独立话题，询问用户：
  - "我检测到N个话题：1)... 2)... 3)...，要分开记录还是合并为一个文件？"
- 如果只有一个话题，直接记录

### 第三步：提取对话内容

从对话历史中提取以下内容：
- **话题名称**：一句话概括这次讨论的核心
- **时间**：对话日期
- **用户说的话**：逐字记录，不删减不总结
- **Claude说的话**：逐字记录，不删减不总结
- **结论/决定**：这次讨论达成的共识或决定

### 第四步：格式化并保存

使用模板 `references/template.md` 格式化。

文件名规则：`NN-话题关键词.md`
- NN：两位序号（01、02、03...）
- 话题关键词：3-6个字的中文概括

保存路径：`C:\Users\liudi\Desktop\LDAI\claude code\<项目名>\开发对话记录\`

### 第五步：确认

保存后告知用户：
- 文件路径
- 话题名称
- 是否有遗漏需要补充

## 记录规则

1. **原话不删减**：用户说的每个字、Claude说的每个字都要记录
2. **不记录工具过程**：代码内容、文件内容、命令输出等不记录
3. **记录工具结论**：如"分析完代码后，发现权重规则是XXX"这类结论要记录
4. **格式忠实**：保持原始的列表、表格等格式，不简化
5. **多话题时**：用户选择分开则分开，选择合并则合并
6. **补录支持**：用户可以要求补录之前的对话（指定话题范围）

## 崩溃恢复

如果对话意外中断（电脑死机、Claude Code崩溃等），未记录的话题不会永久丢失：

1. **原始数据不丢**：Claude Code 会将每条对话持久化到 `~/.claude/` 下的 JSONL 文件，即使崩溃也存在
2. **恢复方式**：用户说"上次对话断了，帮我补录"
3. **自动补录流程**：
   - 我读取 `~/.claude/` 下的 JSONL 文件，找到中断的会话
   - 分析对话内容，自动识别话题边界
   - 对比已有的话题文件，找出未记录的部分
   - 自动创建缺失的话题文件
4. **全程用户只需说一句话**，拆分话题、格式化、保存全由我完成

**防御策略**：长对话中，每完成一个大话题就主动记录一次，不要攒到最后统一记录，降低中断时的损失量。
